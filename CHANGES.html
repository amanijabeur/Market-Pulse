1. Bug Fixes

Anomaly Detection — 0 Results (Root Cause: Insufficient Data)

The Risk/Anomaly tab showed 0 anomalies with "No rows available." because all detectors require a minimum of 5 days of historical data to establish a z-score baseline. With only 4 scraped days, every detector was correctly returning empty — not a code bug, but a data constraint.

DataFrame or Ambiguity Errors (3 locations in main.py)

Multiple pipeline stages used Python's or operator as a None-fallback for DataFrames (e.g., _run_ctx.get("df_full") or load_price_data()). When a DataFrame was already stored in _run_ctx, Python evaluated bool(DataFrame) which raises ValueError. Fixed all 3 occurrences to use explicit if key in _run_ctx checks.

Affected stages: AI Narratives, Anomaly Detection, Historical Fetch

Forecast Charts Showing Flat Lines

The default forecast method was "ema", which by design projects the last rolling average forward as a flat horizontal line. Switched to "linear" (OLS trend extrapolation) in config.py.

2. Date & Timezone Fix — Scraper

Modified scraper.py

The scraper was tagging data with datetime.date.today() — the local Tunisia date. Since Tunisia is UTC+1 and the user sometimes runs the scraper at 3 AM local time (still Sunday in the US), data scraped on Monday local was being tagged as Monday when Yahoo Finance was actually serving Friday's closing prices.

Fix

Added get_us_market_date() function using zoneinfo.ZoneInfo("America/New_York"). Returns the most recent US trading day date, rolling back to Friday on weekends. All data is now tagged with the correct US market date regardless of where the scraper is running.

3. Historical Data Extension

Problem

All intelligence modules (forecasting, signals, anomaly detection, AI briefing) were operating on only 4 days of scraped data. Z-scores, rolling averages, and baselines computed from 4 points are statistically meaningless.

Solution — Three Extended Data Builders

New Functions added to historical_metrics.py
FunctionWhat it buildsUsed by
build_extended_hist_df()Market-wide daily aggregates: combines OHLCV-derived avg_pct / volatility / breadth_pct (500 days) with scraped data. Scraped dates override OHLCV on overlap.Forecasting, AI Narratives
build_extended_df_full()Per-symbol price rows: appends OHLCV-derived daily % Change for each symbol (434 days avg) before the scraper started.Signal Engine, Anomaly Detection, Risk Scores, Opportunity Ranking
_ohlcv_daily_aggregates()Internal helper: loads all 124 cached OHLCV parquet files, computes cross-sectional daily metrics.Called by build_extended_hist_df()
Result: 4 scraped days → 502-day baseline for forecasting. 4 scraped rows per symbol → 434 rows avg for signal z-scores. Data grows daily as scraping continues, gradually replacing OHLCV.

4. Historical Sentiment Extension

VIX as Market-Wide Sentiment Proxy

New finnhub_sentiment.py  |  Modified sentiment.py

Added fetch_vix_history(period="2y") which downloads VIX daily closes from yfinance and converts each reading to a VADER-compatible sentiment score using:

score = -tanh((VIX - 20) / 10)

VIX 10 → +0.76 (bullish)  ·  VIX 20 → 0.00 (neutral)  ·  VIX 30 → -0.76 (bearish)

Finnhub Historical News Sentiment

Added Finnhub's free /company-news API as a historical news source. Headlines are scored with VADER (same approach as the daily scraper). Rate-limited to 60 calls/min to respect the free tier.

Extended Daily Sentiment Average

Added extended_daily_sentiment_avg() to sentiment.py. Combines:

  • Finnhub/VADER history (384 days backfilled in one-time run)
  • VIX-derived scores for any remaining gaps
  • Fresh daily VADER scores from the scraper for recent days

Result: 4-day sentiment baseline → 631-day extended baseline.

7. AI Briefing Improvements

9. Auto-Backfill on Every Run

New stage_auto_backfill() in main.py

Added as the first stage of every main.py run. Detects and fills two classes of gaps silently:

  • Missing price days: Scans the last 30 days for weekday gaps. If the PC was off or the scraper missed a day, fills automatically using yfinance closing prices.
  • Sparse sentiment: If sentiment history has fallen more than 7 days behind the price data, auto-runs a Finnhub backfill for the missing period.

No manual scripts ever need to be run — the pipeline self-heals.

10. OHLCV Fetch Optimization

Modified pipelines/fetch_historical.py

Stage 0 (OHLCV fetch) was running on every main.py call, making 124+ yfinance API checks even when data was already current. Added a sentinel file (.ohlcv_last_fetch) that records the last US market date the fetch ran.

On subsequent runs the same day, the entire stage is skipped in milliseconds instead of running for ~60 seconds. The sentinel resets automatically the next trading day.
